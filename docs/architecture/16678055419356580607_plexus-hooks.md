# Plexus Hook System

## Problem

Schema computation is slow because `plugin_children()` calls `plugin_schema()` directly on each activation, bypassing any caching or interception. When `plexus.schema` is called:

1. `Activation::plugin_schema(&plexus)` is called
2. This calls `plugin_children()` which iterates all 11+ activations
3. Each `a.plugin_schema()` recomputes the full schema from scratch
4. No caching, no hooks, no interception point

The timing shows 5+ seconds total with only 0.15s CPU - but we want to enable caching and other cross-cutting concerns.

## Solution: Uniform Call Routing with Hooks

**Principle: Everything goes through `call()`**

All method invocations, including `schema`, route through `Activation::call()`. This creates a single interception point where hooks can:
- Cache results
- Log/measure calls
- Transform params/results
- Short-circuit with cached values

## Design

### Hook Trait

```rust
// hub-core/src/plexus/hooks.rs

use crate::plexus::{PlexusStream, PlexusError};
use serde_json::Value;

/// Hook for intercepting all Plexus method calls
pub trait PlexusHook: Send + Sync {
    /// Called before a method executes. Return Some to short-circuit.
    fn before_call(
        &self,
        namespace: &str,
        method: &str,
        params: &Value,
    ) -> Option<Result<PlexusStream, PlexusError>> {
        None
    }

    /// Called after a method executes. Can transform the result.
    fn after_call(
        &self,
        namespace: &str,
        method: &str,
        params: &Value,
        result: Result<PlexusStream, PlexusError>,
    ) -> Result<PlexusStream, PlexusError> {
        result
    }
}
```

### Hook Registration

```rust
// In PlexusInner
struct PlexusInner {
    // ... existing fields ...
    hooks: Vec<Arc<dyn PlexusHook>>,
}

impl Plexus {
    /// Add a hook to intercept all calls
    pub fn with_hook(mut self, hook: Arc<dyn PlexusHook>) -> Self {
        Arc::get_mut(&mut self.inner)
            .expect("Cannot add hook: Plexus has multiple references")
            .hooks.push(hook);
        self
    }
}
```

### Wrapping route()

```rust
impl Plexus {
    pub async fn route(&self, method: &str, params: Value) -> Result<PlexusStream, PlexusError> {
        let (namespace, method_name) = self.parse_method(method)?;

        // Before hooks - check for short-circuit (e.g., cache hit)
        for hook in &self.inner.hooks {
            if let Some(result) = hook.before_call(namespace, method_name, &params) {
                return result;
            }
        }

        // Actual call
        let result = self.route_inner(namespace, method_name, params.clone()).await;

        // After hooks - transform/cache result
        let mut result = result;
        for hook in &self.inner.hooks {
            result = hook.after_call(namespace, method_name, &params, result);
        }

        result
    }
}
```

### plugin_children() Through call()

The key change: `plugin_children()` becomes async and routes schema calls through `call()`:

```rust
impl Plexus {
    /// Get child plugin summaries - routes through call() for hook support
    pub async fn plugin_children(&self) -> Vec<ChildSummary> {
        let mut children = Vec::new();

        for (namespace, _activation) in &self.inner.activations {
            // Route through call() - hooks can intercept/cache
            match self.route(&format!("{}.schema", namespace), Value::Null).await {
                Ok(stream) => {
                    // Extract PluginSchema from stream
                    if let Some(schema) = extract_schema_from_stream(stream).await {
                        children.push(ChildSummary {
                            namespace: schema.namespace,
                            description: schema.description,
                            hash: schema.hash,
                        });
                    }
                }
                Err(_) => continue,
            }
        }

        children
    }
}
```

### Cache Hook Implementation

```rust
// hub-core/src/plexus/hooks.rs

use std::sync::RwLock;
use std::collections::HashMap;

/// Cache hook that stores method results by (namespace, method, params)
pub struct CacheHook {
    cache: RwLock<HashMap<(String, String, String), PlexusStream>>,
}

impl CacheHook {
    pub fn new() -> Self {
        Self { cache: RwLock::new(HashMap::new()) }
    }

    fn cache_key(ns: &str, method: &str, params: &Value) -> (String, String, String) {
        (ns.to_string(), method.to_string(), params.to_string())
    }
}

impl PlexusHook for CacheHook {
    fn before_call(&self, ns: &str, method: &str, params: &Value)
        -> Option<Result<PlexusStream, PlexusError>>
    {
        // Only cache schema calls (stateless, deterministic)
        if method != "schema" {
            return None;
        }

        let key = Self::cache_key(ns, method, params);
        self.cache.read().unwrap()
            .get(&key)
            .cloned()
            .map(Ok)
    }

    fn after_call(&self, ns: &str, method: &str, params: &Value, result: Result<PlexusStream, PlexusError>)
        -> Result<PlexusStream, PlexusError>
    {
        // Only cache schema calls
        if method != "schema" {
            return result;
        }

        if let Ok(ref stream) = result {
            let key = Self::cache_key(ns, method, params);
            self.cache.write().unwrap()
                .insert(key, stream.clone());
        }

        result
    }
}
```

### Usage in Builder

pub async fn build_plexus() -> Arc<Plexus> {
    let cache_hook = Arc::new(CacheHook::new());

    Arc::new_cyclic(|weak_plexus: &Weak<Plexus>| {
        Plexus::new()

    })
}

## Considerations

### PlexusStream Cloning

For caching to work, `PlexusStream` needs to be cloneable. Since it's a `Pin<Box<dyn Stream>>`, we may need to:
- Collect stream items into a `Vec` for caching
- Replay from cached `Vec` on cache hit
- Or use a different caching strategy (cache the serialized result)

### Async plugin_children()

Making `plugin_children()` async propagates up:
- `plugin_schema()` for hubs becomes async (calls `plugin_children()`)
- This affects the `Activation` trait

Alternative: Use `block_on` or spawn for the async call within sync context.

### Hook Ordering

Hooks are called in registration order. For cache hooks, order matters:
- `before_call`: First hook to return Some wins (cache should be first)
- `after_call`: All hooks see the result (cache should be last to see final result)

## Future Extensions

- **LoggingHook**: Log all calls with timing
- **MetricsHook**: Count calls, measure latency, export to prometheus
- **AuthHook**: Check permissions before calls
- **RateLimitHook**: Throttle calls per namespace/method
- **TransformHook**: Modify params or results (e.g., add defaults)
